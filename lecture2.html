Lecture 1: Introduction to web development

Client-Server Model

The Client-Server Model is a distributed architecture where clients request services and servers provide them. Clients send requests to servers, which process them and return the results. Clients donâ€™t share resources among themselves but depend on the server. Common examples include email systems and the World Wide Web, where clients (like browsers or email apps) interact with servers to access content or send data. Reference: GeeksforGeeks

Static website Vs Dynamic website

A static website displays the same content to all visitors whereas dynamic websites are always changing and pages have interactive elements that update in real-time to provide a custom experience for each client browser. Static websites have limited functionality. Dynamic websites have more dynamic features where visitors can view customized page content, place items in shopping carts, pay for items, and fill out forms.

Summary

The client-server model separates users (clients) from service providers (servers). HTML tags are the building blocks of web content.
